### 1. ARIMA 销量预测模块

#### 1.1 选用模型
**ARIMA (AutoRegressive Integrated Moving Average)** - 自回归差分移动平均模型

#### 1.2 技术选型理由
- 时间序列预测的标准方法，适用于销售数据
- 能够捕捉趋势、季节性和随机波动
- 支持置信区间输出，便于评估预测不确定性

#### 1.3 模型参数
| 参数 | 值 | 说明 |
|------|-----|------|
| p (AR阶数) | 1-2 | 自回归项数，通过ACF/PACF分析确定 |
| d (差分阶数) | 1 | 使序列平稳的差分次数 |
| q (MA阶数) | 0-2 | 移动平均项数 |
| 预测步长 | 默认30天 | 可配置 |
| 训练数据量 | 最近180天 | 平衡历史信息与计算效率 |

#### 1.4 参数选择方法
- 使用ADF检验确定差分阶数d
- 通过网格搜索选择AIC最小的(p,d,q)组合
- 备选参数: (1,1,0), (1,1,1), (2,1,0), (2,1,1), (1,1,2)

#### 1.5 评估指标
- **RMSE** (均方根误差)
- **MAE** (平均绝对误差)  
- **MAPE** (平均绝对百分比误差)
- **AIC** (赤池信息准则)
- **BIC** (贝叶斯信息准则)

#### 1.6 置信区间
- 默认95%置信区间
- 基于预测的标准误差计算

#### 2.1 选用模型

#### 2.4 聚类特征
| 特征名 | 说明 | 计算方式 |

### 4. Apriori 关联规则挖掘模块

#### 4.1 选用模型
**Apriori 算法** - 频繁项集挖掘与关联规则生成

#### 4.2 技术选型理由
- 经典的关联分析算法
- 规则易于理解和应用
- 适合发现商品组合购买规律

#### 4.3 算法参数
| 参数 | 值 | 说明 |
|------|-----|------|
| min_support | 0.01 (1%) | 最小支持度阈值 |
| min_confidence | 0.5 (50%) | 最小置信度阈值 |
| max_item_length | 3 | 最大项集长度 |

#### 4.4 关联指标
| 指标 | 公式 | 说明 |
|------|------|------|
| **支持度 (Support)** | P(A∪B) | 项集出现的频率 |
| **置信度 (Confidence)** | P(B|A) | 购买A后购买B的条件概率 |
| **提升度 (Lift)** | P(B|A)/P(B) | 关联强度，大于1表示正相关 |
| **杠杆率 (Leverage)** | P(A∪B) - P(A)×P(B) | 衡量关联程度 |
| **确信度 (Conviction)** | (1-P(B))/(1-Confidence) | 规则可靠性 |

## 3 核心算法实现

### 3.1 ARIMA销量预测

#### 3.1.1 算法原理

ARIMA（AutoRegressive Integrated Moving Average）模型是时间序列预测的经典方法。ARIMA模型由三个部分组成：自回归（AR）、差分（I）和移动平均（MA）。

ARIMA模型的一般形式为ARIMA(p,d,q)，其中：
- p：自回归项数，表示使用多少个历史值来预测当前值
- d：差分阶数，表示对序列进行多少次差分使其平稳
- q：移动平均项数，表示使用多少个历史预测误差

#### 3.1.2 参数选择

本系统采用以下参数选择策略：

（1）差分阶数d确定：使用ADF（Augmented Dickey-Fuller）检验判断序列是否平稳，通过差分使序列平稳。

（2）p和q确定：采用网格搜索方法，尝试不同的(p,d,q)组合，选择AIC（Akaike Information Criterion）最小的组合作为最优参数。

#### 3.1.3 实现代码要点

```python
# 参数选择
param_candidates = [(1, d, 0), (1, d, 1), (2, d, 0), (2, d, 1), (1, d, 2)]
for p, d_test, q in param_candidates:
    model = ARIMA(series_clean, order=(p, d_test, q))
    fitted = model.fit(method='innovations_mle')
    if fitted.aic < best_aic:
        best_params = (p, d_test, q)
```

#### 3.1.4 模型评估

系统使用以下指标评估预测效果：
- RMSE（均方根误差）
- MAE（平均绝对误差）
- MAPE（平均绝对百分比误差）
- AIC（赤池信息准则）
- BIC（贝叶斯信息准则）

### 3.2 K-means用户画像聚类

#### 3.2.1 算法原理

K-means算法是一种基于质心的聚类方法。其核心思想是：将n个数据点划分为k个簇，使得每个簇内的数据点与该簇质心的距离平方和最小。

算法步骤如下：
1. 随机选择k个数据点作为初始质心
2. 将每个数据点分配给最近的质心所在的簇
3. 重新计算每个簇的质心
4. 重复步骤2-3，直到质心不再变化或达到最大迭代次数

#### 3.2.2 特征工程

为了构建用户画像，本系统提取以下特征：
- total_orders：订单总数
- total_spend：总消费金额
- avg_order_value：平均客单价
- purchase_frequency：购买频次
- days_since_last：距最近购买天数

在进行聚类之前，使用StandardScaler对特征进行标准化处理，确保各特征对聚类结果的影响均衡。

#### 3.2.3 最优K值确定

采用轮廓系数（Silhouette Score）确定最优聚类数。轮廓系数的计算公式为：

$$s = \frac{b - a}{max(a, b)}$$

其中，a表示样本与同距离，b表示簇其他样本的平均样本与最近簇中样本的平均距离。轮廓系数范围为[-1,1]，越接近1表示聚类效果越好。

#### 3.2.4 实现代码要点

```python
# 寻找最优K值
silhouette_scores = []
for k in range(2, 8):
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    labels = kmeans.fit_predict(scaled_features)
    score = silhouette_score(scaled_features, labels)
    silhouette_scores.append(score)
optimal_k = np.argmax(silhouette_scores) + 2
```

### 3.3 决策树区域分析

#### 3.3.1 算法原理

决策树（Decision Tree）是一种树形结构的分类或回归模型。在本系统中，我们使用决策树回归来预测不同条件下的销售数量。

CART（Classification and Regression Trees）算法使用基尼系数（Gini Impurity）作为分裂准则。对于回归任务，使用均方误差（MSE）作为分裂准则。

#### 3.3.2 输入特征

决策树的输入特征包括：
- 城市（类别型）
- 季节（类别型）
- 品类（类别型）
- 价格区间（数值型）

目标变量为销售数量。

#### 3.3.3 模型参数

为防止过拟合，设置以下参数：
- max_depth=5：最大树深度
- min_samples_split=20：节点分裂最小样本数
- min_samples_leaf=10：叶节点最小样本数

#### 3.3.4 规则提取

从训练好的决策树中提取销售规则，示例规则如下：
- IF 城市=上海 AND 季节=冬季 THEN 羽绒服销量高
- IF 城市=成都 AND 品类=T恤 THEN 春夏季节销量中等

### 3.4 Apriori关联规则

#### 3.4.1 算法原理

Apriori算法是关联规则挖掘的经典算法。其核心思想是：如果一个项集是频繁的，那么它的所有子项集也必须是频繁的。

算法步骤如下：
1. 设定最小支持度阈值，扫描所有交易找出1-频繁项集
2. 基于k-1频繁项集连接生成k-候选项集
3. 剪枝掉包含非频繁子项集的候选项集
4. 扫描交易计算候选项集支持度，找出k-频繁项集
5. 重复步骤2-4，直到不能生成新的频繁项集

#### 3.4.2 关联指标

系统计算以下关联指标：
- **支持度（Support）**：P(A∪B)，表示项集A和B同时出现的概率
- **置信度（Confidence）**：P(B|A)，表示购买A后购买B的条件概率
- **提升度（Lift）**：P(B|A)/P(B)，表示关联强度，大于1表示正相关

#### 3.4.3 实现代码要点

```python
# 生成关联规则
def generate_rules(frequent_itemsets, min_confidence):
    rules = []
    for itemset in frequent_itemsets:
        for i in range(len(itemset)):
            antecedent = itemset[:i] + itemset[i+1:]
            consequent = [itemset[i]]
            confidence = support(itemset) / support(antecedent)
            if confidence >= min_confidence:
                lift = confidence / support(consequent)
                rules.append({
                    'antecedent': antecedent,
                    'consequent': consequent,
                    'support': support(itemset),
                    'confidence': confidence,
                    'lift': lift
                })
    return rules
```

## 5 实验结果与分析

### 5.1 ARIMA预测结果

通过对历史销售数据的分析，系统成功构建了ARIMA预测模型。实验表明：
- 使用最近180天的数据进行训练效果最佳
- 参数(p=1, d=1, q=1)在多数场景下表现良好
- 模型能够捕捉周季节性波动

### 5.2 用户聚类结果

K-means聚类分析识别出4类主要用户群体：
1. **VIP高价值用户**：消费金额高、购买频次高
2. **潜力用户**：当前消费较低但购买频次较高
3. **活跃用户**：购买频次高但客单价一般
4. **沉睡用户**：长时间未购买

### 5.3 区域销售规律

决策树分析揭示了以下区域销售规律：
- 一线城市冬季羽绒服需求量大
- 南方城市T恤销售旺季在春夏
- 高价位商品在东部沿海城市表现更好

### 5.4 商品关联规则

Apriori算法挖掘出以下有价值的关联规则：
- 购买羽绒服的客户往往会购买保暖内衣
- 购买T恤的客户可能会购买牛仔裤
- 运动系列商品存在较强的关联性
